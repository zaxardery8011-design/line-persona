# LINE 影分身 · 白話使用手冊

> 給「不太會寫程式、但想要一隻自己的 LINE AI 分身」的你。
> 全程不用看懂程式碼。看不懂的地方，**直接把這份手冊丟給 AI 叫它幫你做**就好。

---

## 這是什麼？

一隻住在 LINE 裡的 AI 分身。**你餵它你的資料**（你是誰、你的服務、常見問答…），
別人在 LINE 上問它，它就用你的資料回答——等於你的基本資訊，24 小時自動幫你回。

- 資料是**你自己的**（存在你電腦裡，不上傳給別人）
- 大腦**雲端、本地隨你挑**（用 OpenAI / Gemini / Claude，或自己電腦跑的免費 Ollama）
- 門檻低到——**你可以叫 AI 幫你架，自己一行程式都不用碰**

---

## 開始前準備 3 樣東西

1. **一台能上網的電腦**（Windows / Mac 都行）
2. **一個 LINE 官方帳號**（到 [LINE Developers](https://developers.line.biz/) 免費申請 Messaging API，會拿到兩把鑰匙：`Channel access token` 和 `Channel secret`）
3. **一個 AI 大腦的鑰匙**（擇一）：
   - 想最簡單 → 申請一個 OpenAI / Gemini 的 API key
   - 想完全免費、資料不出門 → 在自己電腦裝 [Ollama](https://ollama.com/)（不用 key）

## 方法一【最推薦】：叫 AI 幫你架，你只出一張嘴

如果你電腦有裝 **Claude Code** 或 **Codex**（這兩個都是會幫你打字、改檔、跑指令的 AI 工具），
你根本不用照下面的步驟做，**把下面這段話複製給它就好**：

### 給 Claude Code 用

打開 Claude Code，`cd` 到這個資料夾，然後說：

```
這是一個 LINE AI 分身專案。請幫我完成設定並啟動：
1. 把 .env.example 複製成 .env
2. 填入我的 LINE 鑰匙：access token = 「貼上你的」、secret = 「貼上你的」
3. 我的大腦要用：「OpenAI / Gemini / 本地 Ollama 三選一，貼上你的 key」
4. 幫我把 persona/profile.md 改成：我叫「你的名字」，我是做「你的行業」的，講話「親切/專業…」
5. persona/knowledge.md 幫我放這些常見問答：「貼上你的營業時間、聯絡方式、常見問題」
6. 裝好套件、啟動，並用 Tailscale Funnel 或 ngrok 幫我開一個對外網址
7. 告訴我要把哪個網址貼到 LINE 後台的 Webhook URL
```

它會自己一步步做完，卡住會問你。

### 給 Codex 用

一樣 `cd` 到這個資料夾，把上面那段話貼給 Codex 即可（指令通用）。
Codex 偏好你把需求講清楚一次到位，所以**鑰匙、行業、常見問答先想好再貼**效果最好。

> 💡 這就是這個專案的精神：**會用 AI，就會架。** 你負責「我要什麼」，AI 負責「怎麼做」。

## 方法二：自己照著做（5 步，每步都白話）

### 第 1 步：拿到鑰匙
到 LINE Developers 申請，記下 `Channel access token` 和 `Channel secret`。

### 第 2 步：填鑰匙
把 `.env.example` 複製一份改名成 `.env`，用記事本打開，把鑰匙貼進對應的空格：

```
LINE_CHANNEL_ACCESS_TOKEN=（貼上）
LINE_CHANNEL_SECRET=（貼上）
LLM_BASE_URL=https://api.openai.com/v1
LLM_API_KEY=（貼上你的 AI key）
LLM_MODEL=gpt-4.1-mini
```

### 第 3 步：教它「你是誰、你知道什麼」（見下一章）

### 第 4 步：開機
在資料夾裡開終端機，打：

```
npm install
npm start
```

看到 `line-persona is running` 就是活了。

### 第 5 步：讓 LINE 找得到它（見最後一章）

## 教你的分身：人格 + 知識

只有**兩個檔**要改，用記事本就能改：

### `persona/profile.md` — 它是誰、怎麼講話
寫它的名字、口吻、能回答什麼、**不該回答什麼**。例如：

```
你是「阿明通訊行」的客服小幫手，講話親切、用台灣口語。
只回答跟門市服務有關的問題。
遇到價格殺價、客訴、或你不確定的事，請說「我幫你轉達真人，請稍候」，不要亂編。
```

### `persona/knowledge.md` — 它知道的事
把你要它隨時能回的資訊放這。**就是純文字，想到什麼寫什麼**：

```
營業時間：週一到週六 10:00-21:00，週日公休
地址：台南市…
電話：06-…
常見問題：
Q 手機可以分期嗎？ A 可以，需帶雙證件…
Q 有收購二手機嗎？ A 有，請帶…
```

改完存檔，重開（`npm start`）就生效。

## 資料很多怎麼辦？兩招（這就是你問的「蒸餾」和「掛資料庫」）

`knowledge.md` 適合放「精華」。但如果你有**一大堆雜亂資料**（聊天記錄、產品型錄、上課逐字稿…），別整包塞——會又貴又笨。用這兩招：

### 招式一：蒸餾（雜訊 → 精華）👈 我們的方法

**概念**：先讓 AI 把一大坨雜亂資料「濃縮」成乾淨的重點，再放進 `knowledge.md`。
這樣分身讀的是精華，回得又準又省錢。

**怎麼做（你一樣只出一張嘴）**：把雜亂資料丟給 Claude Code / Codex / 或任何 AI，說：

```
這是我的一堆原始資料（產品型錄/逐字稿/對話記錄）。
請幫我蒸餾成「FAQ 重點清單」，格式是一問一答，去掉重複和廢話，
寫成繁體中文，存到 persona/knowledge.md。
```

> **本專案已內建這支蒸餾腳本！** 直接用：
> ```
> node distill.js 你的原始檔.txt            # 蒸餾後覆蓋 knowledge.md
> node distill.js 你的原始檔.txt --append    # 蒸餾後附加到 knowledge.md
> ```
> 它會用你 `.env` 裡設好的同一個 AI（雲端或本地都行）把資料濃縮成 FAQ，寫進 `persona/knowledge.md`。
> 資料常更新時，改完原始檔再跑一次即可。

### 招式二：掛資料庫（進階，資料多到塞不下時）

當你的知識多到**幾百頁、塞不進一次對話**時，就不能再「全部讀進去」，
要改成「**需要時才去查相關段落**」——這就是俗稱的 RAG / 知識庫。

**改的位置**：`src/brain.js` 就是「大腦插孔」。預設是把整份 `knowledge.md` 讀進去；
你（或你的 AI）只要在這裡多加一步「先去資料庫查最相關的幾段，再餵給模型」就行。

**怎麼做**：把需求丟給 Claude Code / Codex：

```
我的知識資料太多了，請幫我把 src/brain.js 從「整包讀進去」
改成「先用向量檢索查最相關的 3-5 段再回答」。
資料庫請用最輕量、不用另外架伺服器的方案（例如本地檔案型向量庫）。
```

> 為什麼預設不做這個？因為**大多數人用不到**——一般個人/小店的資訊，`knowledge.md` 就夠了。
> 我們刻意保持最簡單，等你真的需要再升級。

## 換大腦：雲端 or 本地

只改 `.env` 三行，不用動程式：

| 想要 | 怎麼填 |
|---|---|
| 最方便（雲端 OpenAI） | `LLM_BASE_URL=https://api.openai.com/v1`、填 key、`LLM_MODEL=gpt-4.1-mini` |
| 用 Gemini / Claude | 透過 OpenAI 相容的 gateway，填它給的網址和 key |
| 完全免費、資料不出門（本地） | 裝 Ollama，`LLM_BASE_URL=http://localhost:11434/v1`、key 留空、`LLM_MODEL=llama3.1` |

## 讓 LINE 找得到它（Webhook）

你的分身在自己電腦上跑，但 LINE 在外面——要給它一個**對外網址**搭橋：

**最簡單（免費、固定網址）**：裝 Tailscale，然後：

```
tailscale funnel 3000
```

它會給你一個 `https://你的機器.xxx.ts.net` 開頭的網址。
到 LINE 後台的 **Webhook URL** 填上去，**後面記得加 `/webhook`**：

```
https://你的機器.xxx.ts.net/webhook
```

按「驗證」變綠燈，就通了。打開 LINE 加你的官方帳號，傳訊息試試看 🎉

> 也可以用 `ngrok http 3000`，但免費版每次重開網址會變，要再貼一次。

## 常見問題

**Q：我完全不會程式，真的能用嗎？**
A：能。用「方法一」，把指令貼給 Claude Code 或 Codex，你只要會複製貼上 + 回答它的問題。

**Q：我的資料會被誰看到？**
A：資料存在你自己電腦的 `persona/` 資料夾。用本地 Ollama 的話，**連大腦都不出門**，最隱私。

**Q：要花錢嗎？**
A：LINE 官方帳號基本免費；本地 Ollama 免費；只有用雲端 AI（OpenAI 等）才按用量算錢，量小很便宜。

**Q：它會亂講話嗎？**
A：在 `profile.md` 把「不確定就說轉真人、不要亂編」寫清楚，能大幅降低。這是你能控制的。
